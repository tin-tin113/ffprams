# FFPRAMS Workflow (Simple Version)

This guide is written for non-technical users.

## 1) Daily Main Flow

1. Open FFPRAMS and login.
2. Check if your Program, Resource Type, and Purpose are ready.
3. Add or update beneficiaries.
4. Admin reviews and approves beneficiary list.
5. Create the distribution event.
6. Add allocations (single, bulk, or direct assistance).
7. Print the distribution list and collect signatures.
8. Mark each result as:
   - Received
   - Not Received
9. Admin marks event as Completed.
10. Open Reports and review totals and gaps.

## 2) Extra Requirements for Financial Events

Before starting a financial event, fill these details:

1. Legal basis (type, reference number, date)
2. Fund source
3. Liquidation details
4. FARMC details if needed

Important:

- Event cannot start if required details are missing.
- Event cannot be completed until liquidation is Verified.

## 3) Audit Log

Use Audit Log to see system actions.

You can answer:

1. Who did the action?
2. What action was done?
3. When did it happen?
4. What changed before and after?

## 4) Common Action Names (Easy Meaning)

1. login: User signed in
2. logout: User signed out
3. profile updated: User changed profile info
4. password updated: User changed password
5. created: New record added
6. updated: Existing record edited
7. deleted: Record removed
7. account deleted: User account removed

## 5) If Something Looks Wrong

1. Open Audit Log.
2. Filter by person, action, and date.
3. Open details and compare Before vs After.
4. Report findings to Admin.
